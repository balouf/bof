Factor Tree¶

The factortree module contains the core engine of the Bag of Factors package.

class bof.factortree.FactorTree(corpus=None, auto_update=False, preprocessor=None, n_range=5, filename=None, path='.')[source]¶

Maintain a tree of factor of a given corpus.

Parameters

  • corpus (list of str, optional) – Corpus of documents to decompose into factors.

  • auto_update (bool, optional) – When processing external texts, should the tree be updated (TO BE MOVED WITH THE FACTORS MODULE WHEN CREATED).

  • preprocessor (callable) – Preprocessing function to apply to texts before adding them to the factor tree.

  • n_range (int or None, optional) – Maximum factor size. If None, all factors will be extracted.

count¶

Keep for each factor a dict that tells for each document (represented by its index) the number of occurences of the factor in the document.

Type

list of dict

edges¶

Keep for each factor a dict that associates to each letter the corresponding factor index in the tree (if any).

Type

list of dict

corpus_list¶

The corpus list.

Type

list of srt

corpus_dict¶

Reverse index of the corpus (corpus_dict[corpus_list[i]] == i).

Type

dict of str -> int

factor_list¶

The factor list.

Type

list of srt

factor_dict¶

Reverse index of the factors (factor_dict[factor_list[i]] == i).

Type

dict of str -> int

self_factors¶

Number of unique factors for each text.

Type

list of int

n¶

Number of texts.

Type

int

m¶

Number of factors.

Type

int

Examples

Build a tree from a corpus of texts,limiting factor size to 3:

>>> corpus = ["riri", "fifi", "rififi"]
>>> tree = FactorTree(corpus=corpus, n_range=3)

List the number of unique factors for each text:

>>> tree.self_factors
[7, 7, 10]

List the factors in the corpus:

>>> tree.factor_list
['', 'r', 'ri', 'rir', 'i', 'ir', 'iri', 'f', 'fi', 'fif', 'if', 'ifi', 'rif']

bof.jc.common_factors(txt, tree, update_tree=False)[source]¶

Compute the number of common factors between a text and a tree corpus.

  • Checks if the text belongs to the corpus first;

  • Update the tree if update_tree is True.

Parameters

  • txt (str) – Text to analyze

  • tree (FactorTree) – Factor tree of the corpus.

  • update_tree (bool) – Update tree if txt is not in it?

Returns

  • factors (list of int) – List of numbers of unique common factors.

  • self_factor (int) – Number of unique factors of the reference text.

Examples

>>> tree = FactorTree(["riri", "fifi", "rififi"])
>>> common_factors("riri", tree)
([8, 2, 4], 8)
>>> common_factors("rafi", tree)
([3, 4, 5], 11)
>>> tree.corpus_list
['riri', 'fifi', 'rififi']
>>> common_factors("fifi", tree, update_tree=True)
([2, 8, 8], 8)
>>> tree.corpus_list
['riri', 'fifi', 'rififi']
>>> common_factors("fari", tree, update_tree=True)
([4, 3, 5, 11], 11)
>>> tree.corpus_list
['riri', 'fifi', 'rififi', 'fari']

bof.jc.joint_complexity(txt, tree, balance=0.5, update_tree=False)[source]¶

Computes Joint Complexity between a text and a corpus.

Parameters

  • txt (str) – Text to analyze

  • tree (FactorTree) – Factor tree of the corpus.

  • balance (:py:class:`float`in range [0.0, 1.0]) – Large values will disregard the size of corpus elements..

  • update_tree (bool) – Update tree if txt is not in it?

Returns

The Joint Complexity distances between the text and corpus.

Return type

list of float

Examples

>>> tree = FactorTree(["riri", "fifi", "rififi"], n_range=None)
>>> joint_complexity("fififi", tree)
[0.8888888888888888, 0.3333333333333333, 0.4444444444444444]
>>> joint_complexity("rifi", tree)
[0.7142857142857143, 0.5, 0.375]

With low balance, rifi is closer to fifi than rififi because rififi is longer.

>>> joint_complexity("rifi", tree, balance=0.0)
[0.6666666666666666, 0.4, 0.5454545454545454]

With high bbalance, rifi is closer to rififi than fifi because rififi may be longer but it contains rifi.

>>> joint_complexity("rifi", tree, balance=1.0)
[0.75, 0.5714285714285714, 0.0]
